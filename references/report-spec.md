# 深度分析报告结构规范（Phase 2 写作用）

报告 = 元数据 + 11 模块 + 「核心争议与反方观点」专章 + 5 XML schema。目标 15k 中文字。

## 报告元数据（开头表格）

公司全称 / 代码 / 行业 / 注册地总部 / 当前股价 / 市值 / 流通股本 / 52 周区间 / YTD / **投资评级** / **12 个月目标价** / **置信度** / 数据可信度（DS-5 条数 + DS-3 以下占比）。

## 11 模块（M-01 ~ M-11）

| 模块 | 内容 | 正文须消化的红队自问 |
|------|------|--------------------|
| M-01 执行摘要 | 一段话讲清「核心争议是什么」，不堆「创纪录」字眼 | 先点明真正的争议（不是「业务好不好」而是「配不配当前估值」） |
| M-02 公司解析 | 业务板块拆分表（营收/增速/利润率）、商业模式飞轮、并购史 | 「多元化」是否真不相关；交叉销售飞轮的证据缺口 |
| M-03 财务深度 | 3 年损益表、营收质量、毛利/现金流、（金融公司：NIM/资金成本/信用质量/资本/ROTCE） | 净利润异动的会计还原；营收质量分层；高毛利/NIM 是护城河还是风险；信用/减值口径；公允价值对单季利润的放大 |
| M-04 估值 | 当前倍数、估值锚（行业对应公式）、相对+绝对+情景三方交叉、估值结论表 | 估值核心豪赌变量（市场已隐含什么预期）；目标价为何偏离卖方共识 |
| M-05 行业 | 赛道、竞争格局、监管、周期位置 | 商业模式的「样样松」反命题 |
| M-06 增长 | 用户/营收/产品增长、增长「质」的追问 | 增长的「质」比「量」的追问 |
| M-07 情景 | 牛/基准/熊三档 + 概率 + 各档目标价 + 概率加权 | 情景分析的不对称性检验 |
| M-08 投资建议 | 评级 + 目标价 + 置信度 + 操作建议（建仓/加仓/止损触发） | — |
| **核心争议与反方观点** | M-08 之后的专章，见下节 | 收口决定结论的 5-6 个争议 + 报告自检 |
| M-09 ESG | E/S/G 三维 + 数据安全 | — |
| M-10 管理层 | CEO 背景/薪酬、内部人持股与交易、股权结构 | — |
| M-11 附录 | 数据来源与可信度表、关键数据缺口、免责声明 | — |

## 红队机制：自我证伪 + 核心争议专章

红队 = 主动唱反调、反叙事偏误的免疫系统。**生成阶段**必须对每个多头叙事做自我证伪追问（≥15 个，每个落在具体的会计科目 / 估值假设 / 数据缺口上）。**呈现阶段**分两层，不再用内联 `🔴 RT-XX` callout：

**第一层 · 融入正文**：大部分红队观点用分析师自己的叙述口吻写进对应模块（「但需警惕…」「这里有个口径问题…」），保持单一声音。见上表「正文须消化的红队自问」。

**第二层 · 核心争议专章**：把真正决定投资结论的 5-6 个争议，收口到 M-08 投资建议之后的「核心争议与反方观点（Key Debates）」专章。每条用三段式：

```
### 争议 N：标题
- 多头：……
- 空头：……
- 我们的处理：……
```

专章须以一节「自检：对本报告自身的偏误审查」收尾（锚定偏误 / 参数保守度 / 是否低估管理层）。典型 5-6 条争议（按 SOFI 案例）：估值核心豪赌变量、关键风险科目的口径争议、最大未核实软肋（数据缺口→置信度）、增长里的易碎收入、赔率不对称性。

⚠️ **禁止**满篇 `🔴 RT-XX` 内联红框——打断阅读、像 track-changes 草稿，不符合中金/Goldman 研报风格（2026-05-16 Master 反馈的正式升级）。

## 5 个 XML schema（报告结尾，便于结构化提取）

用 ```xml 代码块包裹，5 个依次为：

1. `<company_profile>` — 名称/代码/行业/总部/业务板块/用户数
2. `<financial_summary>` — 3 年损益、关键行业指标、信用/质量指标
3. `<valuation>` — 当前倍数、COE 参数、三方估值结果、市场隐含预期、目标价
4. `<scenario_analysis>` — 牛/基准/熊三档（概率 + 目标价 + driver）+ 概率加权
5. `<investment_rating>` — 评级/目标价/上行空间/置信度/周期/关键风险/上调触发/仓位建议

## 质量门槛

- 15k 中文字 / 11 模块全 / 核心争议专章 ≥ 5 条争议 / XML 5 个全
- DS-3 以下数据占比 < 15%（做空事件、卖方观点等不可避免的 DS-3 要逐条标注）
- 目标价三方交叉验证收敛
- 置信度与数据缺口诚实挂钩
