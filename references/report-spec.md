# 深度分析报告结构规范（Phase 2 写作用）

报告 = 元数据 + 11 模块 + 15 红队节点（内嵌）+ 5 XML schema。目标 15k 中文字。

## 报告元数据（开头表格）

公司全称 / 代码 / 行业 / 注册地总部 / 当前股价 / 市值 / 流通股本 / 52 周区间 / YTD / **投资评级** / **12 个月目标价** / **置信度** / 数据可信度（DS-5 条数 + DS-3 以下占比）。

## 11 模块（M-01 ~ M-11）

| 模块 | 内容 | 内嵌红队节点 |
|------|------|------------|
| M-01 执行摘要 | 一段话讲清「核心争议是什么」，不堆「创纪录」字眼 | RT-01 |
| M-02 公司解析 | 业务板块拆分表（营收/增速/利润率）、商业模式飞轮、并购史 | RT-02, RT-03 |
| M-03 财务深度 | 3 年损益表、营收质量、毛利/现金流、（金融公司：NIM/资金成本/信用质量/资本/ROTCE） | RT-04~08 |
| M-04 估值 | 当前倍数、估值锚（行业对应公式）、相对+绝对+情景三方交叉、估值结论表 | RT-09, RT-10 |
| M-05 行业 | 赛道、竞争格局、监管、周期位置 | RT-11 |
| M-06 增长 | 用户/营收/产品增长、增长「质」的追问 | RT-12 |
| M-07 情景 | 牛/基准/熊三档 + 概率 + 各档目标价 + 概率加权 | RT-13 |
| M-08 投资建议 | 评级 + 目标价 + 置信度 + 操作建议（建仓/加仓/止损触发） | RT-14, RT-15 |
| M-09 ESG | E/S/G 三维 + 数据安全 | — |
| M-10 管理层 | CEO 背景/薪酬、内部人持股与交易、股权结构 | — |
| M-11 附录 | 数据来源与可信度表、关键数据缺口、免责声明 | — |

## 15 红队节点（RT-01 ~ RT-15）

红队节点 = 主动唱反调的免疫系统。每个 RT 必须落在**一个具体的会计科目 / 估值假设 / 数据缺口**上，不能是对管理层叙事的复述。用 `> 🔴 **RT-XX｜标题**：...` 内嵌到对应模块。

参考分布（按 SOFI 案例）：
- RT-01 执行摘要的叙事陷阱（先点明真正的争议）
- RT-02 业务「多元化」是否真的不相关
- RT-03 交叉销售飞轮的证据缺口
- RT-04 净利润同比异动的会计还原
- RT-05 营收质量分层（高质量 vs 易断崖收入）
- RT-06 高毛利/高 NIM 是护城河还是风险敞口
- RT-07 信用/减值数据的口径警报
- RT-08 公允价值计量对单季利润的放大
- RT-09 估值的核心豪赌变量（市场已隐含什么预期）
- RT-10 目标价为何偏离卖方共识
- RT-11 商业模式的「样样松」反命题
- RT-12 增长的「质」比「量」的追问
- RT-13 情景分析的不对称性检验
- RT-14 本评级最大的未核实软肋（数据缺口 → 置信度）
- RT-15 对报告自身的反向审查（锚定偏误 / 参数保守度 / 是否低估管理层）

## 5 个 XML schema（报告结尾，便于结构化提取）

用 ```xml 代码块包裹，5 个依次为：

1. `<company_profile>` — 名称/代码/行业/总部/业务板块/用户数
2. `<financial_summary>` — 3 年损益、关键行业指标、信用/质量指标
3. `<valuation>` — 当前倍数、COE 参数、三方估值结果、市场隐含预期、目标价
4. `<scenario_analysis>` — 牛/基准/熊三档（概率 + 目标价 + driver）+ 概率加权
5. `<investment_rating>` — 评级/目标价/上行空间/置信度/周期/关键风险/上调触发/仓位建议

## 质量门槛

- 15k 中文字 / 11 模块全 / RT 15 个全 / XML 5 个全
- DS-3 以下数据占比 < 15%（做空事件、卖方观点等不可避免的 DS-3 要逐条标注）
- 目标价三方交叉验证收敛
- 置信度与数据缺口诚实挂钩
